# AniTrack UI Design Guidelines

AniTrack 的界面方向统一为 `Mochi AniTrack`：浅色、现代、二次元、马卡龙、可爱但不幼稚、轻动效。不要做深色模式，不直接照搬 Jellyfin；只参考它的信息架构：分组清晰、主页面简洁、复杂编辑进入独立界面。

## Layout

- 主页面优先展示卡片、列表和状态摘要，不直接堆复杂表单。
- 新增、编辑、配置类操作默认使用弹窗或抽屉。
- 全站优先采用“个人中心式”布局：左侧状态名片 + 导航，右侧居中内容区；不要把内容全屏后台式铺满。
- 左侧栏必须拆成独立卡片：Logo/标题、实时状态、媒体导航、系统导航/账号操作分开呈现，卡片之间保留留白。
- 页面顶部不使用通用大标题横幅；每页只保留必要的页内工具条、筛选和主操作。
- 每个页面必须先显示统一的页面标题卡片，主内容整体下移；标题卡只放页面名、短说明和轻量状态，不做后台式大横幅。
- 设置页按功能分组：基础、自动选择、下载器、搜索源、媒体库、定时器、维护。
- 设置页、搜索源、下载器、定时器等复杂配置必须使用“摘要列表 + 弹窗编辑”，主页面只展示名称、类型、状态和关键摘要。
- 下载器、搜索源、新番 RSS 源必须使用一致的摘要列表：拖拽排序、启用开关、编辑弹窗、简短状态摘要。
- 列表页的空状态必须居中显示，图片和文字作为整体居中。
- 大量操作按钮必须分组展示，避免混成一排。
- 控制台以主控台卡片和任务看板呈现，不直接暴露内部 processor 概念。
- 控制台右侧优先展示持久化“最近操作”，不要重复展示任务列表；RSS 状态放到侧栏实时状态卡或轻量状态区。
- 媒体页使用统一卡片流；封面、标题、状态、可观看、评分优先显示，长详情放抽屉。
- 日历使用“周一到周日 Tab + 当天更新卡片”结构，不使用七列窄周历；同一作品同日多集只显示最新一集。

## Components

- Toggle 统一为开关在前、说明在后。
- 数字配置默认使用普通输入框，只有明确需要步进时才用 stepper。
- 搜索源、下载器、优先级列表使用拖拽排序，不让用户手填排序数字。
- 文件选择按钮、普通按钮、下拉操作按钮必须使用同一套尺寸、圆角、颜色和 hover 状态。
- 支持上传时，文件上传和文件夹上传必须并列为统一按钮样式；目录上传要保留相对路径，便于自动识别作品和集数。
- 彩色按钮文字必须清晰；大面积操作优先使用 plain 风格。
- 危险操作使用 danger，并且必须有二次确认。
- 按钮必须可交互地表现 hover/active，不做纯静态样板。
- 表格只放核心字段；长路径、链接和调试信息放展开区、弹窗或日志。
- 任务操作必须走统一任务接口：单条任务提供取消/暂停/继续/重试/清理中适用的操作，任务列表顶部提供批量取消、暂停、继续、重试失败和清理。
- 简单登录保护为单管理员模型；账号入口放设置页，侧栏只保留简短账号操作，不做拟人化用户登录面板。

## Visual Style

- 主色为浅粉，辅色为浅蓝、薄紫、薄黄、薄荷绿，避免全站单一粉色。
- 卡片使用白底、浅粉边框、柔和阴影，中大圆角。
- 动效只用于 hover、选中、加载状态，保持轻量；允许卡片轻浮起、按钮轻果冻感、进度条柔和渐变。
- Toast 统一显示在右上角堆叠，不使用页面中间遮挡式消息条。
- Toast 使用白色胶囊卡片和轻量 emoji/icon；最多显示少量队列，避免遮挡主操作区。
- 用户触发的长耗时操作必须异步提交任务并立刻返回；按钮可短暂 loading，但不能在用户关闭抽屉/弹窗后又自动弹回。
- 全局操作反馈必须进入右上角 Toast：提交成功、失败、任务完成、任务失败都要有明确提示，文案可少量使用 emoji 增强识别。
- 最近操作是审计流，不是无限日志；页面展示必须分页或固定高度滚动，默认只看最近少量记录。
- 最近操作后端最多保留 100 条；超出时自动删除最早记录，控制台必须提供清理按钮。
- 复杂数据行优先做紧凑列表，详细信息放展开区、弹窗或抽屉。
- 不依赖外部 CDN 字体、Tailwind CDN 或 FontAwesome；所有视觉必须在项目内可构建。

## Static Capture Workflow

- 如果要让其他 AI 修改 UI，优先使用当前后台的 SingleFile 静态快照，而不是重新手写一份不一致的 mockup。
- 快照统一放在 `docs/ui-mockups/live-captures-YYYYMMDD/`，并写 `manifest.md` 说明页面和状态。
- 当前可参考的真实快照目录是 `docs/ui-mockups/live-captures-20260626/`。
- Vue 回灌时必须对照快照的真实结构、间距、按钮状态和弹窗状态，不要只参考配色。


一、 核心配色调用 (Mochi Color Palette)
  进行任何页面新增或按钮修改时，请统一使用 style-part-1.css 中定义的 CSS 变量，确保不脱离主题风格：
   * 甜桃粉 (--ani-pink): #FF8DA1 (主色调，用于代表活力、在看、最爱)
   * 深蜜桃 (--ani-pink-deep): #FF5E7E (悬浮点击高亮、重要删除、高位警示)
   * 汽水蓝 (--ani-blue): #82D9ED (信息色，用于同步、扫描、想看)
   * 蓝莓紫 (--ani-purple): #C79FE5 (特殊色，用于等级标签、定时调度)
   * 薄荷绿 (--ani-mint): #8EE4D5 (成功色，用于已看毕、校验成功)
   * 深紫灰 (--ani-dark): #5C525D (文本主色，千万不要用 #000 或 #333！用深紫灰会极其温和护眼)
   * 樱花粉描边 (--ani-line): #FFEBEF (全局所有的卡片、按钮、边框，请使用此粉色，代替原有的白灰线)

  二、 排版与对齐规范 (Grid & Alignments)
   1. 统一间距：网格间距请一律使用 gap: 16px，大外边距使用 padding: 20px 24px。
   2. 工具栏对齐：工具栏中的输入框和操作按钮，必须通过 .toolbar-spacer 或 Flex 布局的弹性自适应保持左右两侧的严密对齐。
   3. 功能堆砌规整化：如果有多个控制按钮在同一个面板中，必须使用卡片式分组
      (.maintenance-group-card)，为每个按钮配合独立的标题与一行功能释义，拒绝无序的平铺堆叠。

  三、 二次元卡片与海报规范 (Card & Covers)
   1. 海报纵横比：海报外容器 .cover 统一使用 aspect-ratio: 2 / 3 与 height: auto !important，确保不丢失画面。
   2. 猫耳装饰 (.cute-cat-ears)：重要的页面卡片顶部，可以通过添加猫耳 CSS 伪元素，营造俏皮可爱的阿宅风格。
   3. Q弹触感 (Jelly Motion)：所有的按钮、卡片、侧边按钮，必须加上 .hover-jelly 或对应的果冻弹性动效，增强物理软糖感。
