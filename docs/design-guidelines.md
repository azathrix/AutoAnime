# AniTrack UI Design Guidelines

AniTrack 的界面方向是浅色、现代、二次元、可爱、轻动效。不要做深色模式，也不要直接照搬 Jellyfin；只参考它的信息架构：分组清晰、主页面简洁、复杂编辑进入独立界面。

## Layout

- 主页面优先展示卡片、列表和状态摘要，不直接堆复杂表单。
- 新增、编辑、配置类操作默认使用弹窗或抽屉。
- 设置页按功能分组：基础、自动选择、下载器、搜索源、媒体库、定时器、维护。
- 列表页的空状态必须居中显示，图片和文字作为整体居中。
- 大量操作按钮必须分组展示，避免混成一排。

## Components

- Toggle 统一为开关在前、说明在后。
- 数字配置默认使用普通输入框，只有明确需要步进时才用 stepper。
- 搜索源、下载器、优先级列表使用拖拽排序，不让用户手填排序数字。
- 彩色按钮文字必须清晰；大面积操作优先使用 plain 风格。
- 危险操作使用 danger，并且必须有二次确认。

## Visual Style

- 允许浅粉、浅蓝、浅紫作为点缀，但不能让页面变成单一色块。
- 卡片圆角保持中等，避免过度圆润。
- 动效只用于 hover、选中、加载状态，保持轻量。
- 复杂数据行优先做紧凑列表，详细信息放展开区、弹窗或抽屉。

## Current Candidates

- Sakura Panel：浅粉/浅蓝、柔和卡片、轻动效，适合媒体页面。
- Clean Anime：克制、清晰、信息密度更高，适合设置页。
- Glass Pop：半透明面板，适合发现页和媒体卡片流。
