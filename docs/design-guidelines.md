# AniTrack UI Design Guidelines

AniTrack 的界面方向统一为 `Mochi AniTrack`：浅色、现代、二次元、马卡龙、可爱但不幼稚、轻动效。不要做深色模式，不直接照搬 Jellyfin；只参考它的信息架构：分组清晰、主页面简洁、复杂编辑进入独立界面。

## Layout

- 主页面优先展示卡片、列表和状态摘要，不直接堆复杂表单。
- 新增、编辑、配置类操作默认使用弹窗或抽屉。
- 设置页按功能分组：基础、自动选择、下载器、搜索源、媒体库、定时器、维护。
- 列表页的空状态必须居中显示，图片和文字作为整体居中。
- 大量操作按钮必须分组展示，避免混成一排。
- 控制台以主控台卡片和任务看板呈现，不直接暴露内部 processor 概念。
- 媒体页使用统一卡片流；封面、标题、状态、可观看、评分优先显示，长详情放抽屉。

## Components

- Toggle 统一为开关在前、说明在后。
- 数字配置默认使用普通输入框，只有明确需要步进时才用 stepper。
- 搜索源、下载器、优先级列表使用拖拽排序，不让用户手填排序数字。
- 彩色按钮文字必须清晰；大面积操作优先使用 plain 风格。
- 危险操作使用 danger，并且必须有二次确认。
- 按钮必须可交互地表现 hover/active，不做纯静态样板。
- 表格只放核心字段；长路径、链接和调试信息放展开区、弹窗或日志。

## Visual Style

- 主色为浅粉，辅色为浅蓝、薄紫、薄黄、薄荷绿，避免全站单一粉色。
- 卡片使用白底、浅粉边框、柔和阴影，中大圆角。
- 动效只用于 hover、选中、加载状态，保持轻量；允许卡片轻浮起、按钮轻果冻感、进度条柔和渐变。
- 复杂数据行优先做紧凑列表，详细信息放展开区、弹窗或抽屉。
- 不依赖外部 CDN 字体、Tailwind CDN 或 FontAwesome；所有视觉必须在项目内可构建。

## Static Capture Workflow

- 如果要让其他 AI 修改 UI，优先使用当前后台的 SingleFile 静态快照，而不是重新手写一份不一致的 mockup。
- 快照统一放在 `docs/ui-mockups/live-captures-YYYYMMDD/`，并写 `manifest.md` 说明页面和状态。
- 当前可参考的真实快照目录是 `docs/ui-mockups/live-captures-20260626/`。
- Vue 回灌时必须对照快照的真实结构、间距、按钮状态和弹窗状态，不要只参考配色。
