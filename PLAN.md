# AutoAnime 页面与媒体模型优化计划

## Summary

本轮基于已还原的旧项目继续小步优化，不再推倒重写。目标是把产品从“Mikan/RSS/队列暴露型工具”收敛成“新番追更 + 通用媒体库”的界面与数据语义。

实现顺序固定为：

1. 先重写 `PLAN.md`，用本计划替换旧计划。
2. 先做 UI/入口收敛和无风险删除。
3. 再做媒体页面与详情页改造。
4. 最后做通用集数资源结构、下载器优先级和设置结构调整。

每个阶段完成后提交；需要测试前运行 `package-clean.bat`。

## Key Changes

### 1. 删除旧导入流程

- 删除左侧“导入”页面入口、移动端入口、导入页面模板和相关前端状态函数。
- 删除旧导入 API：
  - `/api/import/local/preview`
  - `/api/import/torrent/preview`
  - `/api/import/local/commit`
  - `/api/import/torrent/commit`
  - `/api/library/import`
- 删除或停用 `import_service.py` 相关引用。
- 保留底层媒体库表，不保留旧导入业务逻辑。
- 后续番剧/电影/电视剧导入按新向导重做，不兼容旧流程。

### 2. 控制台优化

- 移除“按集运行态”区域。
- 队列详情右侧去掉 `全部 / 新番 / 待调度` 等筛选按钮。
- 控制台默认显示“未选择队列”的空状态，避免活跃队列为空时右侧空白。
- 定时任务保留在控制台内：
  - RSS 扫描支持开关、间隔配置。
  - 恢复调度支持开关、间隔配置。
- 设置页删除：
  - 自动刷新间隔。
  - 追更自动下载。
  - 自动下载唯一匹配/按优先级选择等旧开关。
- 新增独立“日志”页面：
  - 左侧导航增加“日志”按钮。
  - 原控制台底部“日志”和“维护”迁移到该页面。
  - 功能保持：日志查看、清空、导出、维护动作。
  - 控制台不再显示日志/维护块。

### 3. 新番页面

- 新番卡片按 `entry` 显示，不按作品折叠。
  - 优先用 Bangumi subject 区分条目。
  - 同一作品不同季/篇章显示为多张卡。
- 卡片只保留核心信息：
  - 标题。
  - 季/篇章/部分信息。
  - `可观看 4 集`。
  - 最近一周有更新时显示 `已更新`。
- 删除卡片进度条、底部状态文案、多余发布数/本地数标签。
- 删除旧状态筛选行：`全部 / 可观看 / 处理中 / 需处理 / 未缓存`。
- 搜索栏下方新增可折叠高级过滤：
  - 年份/月。
  - 标签。
  - 国家/地区。
  - 类型。
  - 数据来源字段来自 Bangumi/TMDB/手动设置。
- 搜索栏最右侧新增“添加 RSS 订阅”按钮。
  - 弹出 RSS 配置页。
  - 支持添加任意 RSS。
  - 订阅类型字段先只支持 `mikan`。

### 4. 新番详情页

- 删除详情页旧动作：
  - 刷新元数据。
  - 生成 NFO。
  - 补全条目。
  - 隐藏误识别。
  - 自动下载。
  - 补全。
- 详情页改为两个主要 Tab：
  - `信息`
  - `集数资源`
- `信息` Tab 显示：
  - Bangumi ID。
  - TMDB ID。
  - 标题/别名。
  - 年份/月。
  - 媒体类型：动画/电影/电视剧。
  - 标签。
  - 国家/地区。
  - 追番状态。
  - 归档按钮。
- “归档”只关闭新番追番状态。
  - 新番页不再显示。
  - 番剧库仍显示该动画。
- `集数资源` Tab 显示通用集数配置：
  - 集数。
  - 当前选中资源。
  - 字幕组。
  - 分辨率。
  - 语言。
  - 字幕类型：内嵌/内封/外挂。
  - 字幕文件信息。
  - 是否已下载。
  - 本地文件路径。
- 不再显示 RSS 原始发布列表、下载任务、PikPak 任务、任务产物等任务细节。

### 5. 通用集数资源模型

新增或重构为“作品/条目/集数配置”模型：

- `works`：作品根。
- `entries`：具体季/篇章/电影/电视剧季。
- `episodes`：集数。
- `episode_resources`：某一集的候选或选中资源。
- `episode_subtitles`：外挂字幕信息。

规则：

- RSS 原始记录不作为长期核心数据。
- RSS 扫描只负责解析并生成标准 `episode_resources`。
- 资源来源可以是 Mikan、手动磁链、本地上传、未来爬虫。
- 自动选集只负责给某集选出一个当前资源。
- 没有精确匹配时也自动选择第一个资源，保证每集有一个默认资源。
- 下载成功后只更新对应集的“已下载/本地路径/字幕信息”。
- 任务失败只体现在任务队列，不污染媒体详情页。

### 6. 番剧、电影、电视剧页面

- 新增或改造三个统一媒体库页面：
  - 番剧：`anime`
  - 电影：`movie`
  - 电视剧：`tv`
- 页面结构与新番页一致：
  - 搜索栏。
  - 可折叠高级过滤。
  - 卡片列表。
  - 详情抽屉。
- 番剧页按钮：
  - `导入`
  - `添加`
- 电影页、电视剧页同样使用 `导入 / 添加`。
- 导入向导先做 UI 骨架，不接旧 API：
  1. 选择本地目录/文件上传入口。
  2. 匹配作品信息。
  3. 匹配集数与字幕。
  4. 收录到对应媒体类型。
- 添加向导：
  1. 输入 Bangumi ID / TMDB ID / 作品名 / 磁力链接。
  2. 匹配作品。
  3. 匹配集数与字幕。
  4. 收录到对应媒体类型。
- 浏览器本地目录选择按“上传器”设计；NAS 服务端路径输入不作为第一版主流程。

### 7. 设置页重构

- 删除系统 Tab。
- 保留代理设置。
- 补全设置改为一个 toggle：`补全本季`。
- 自动生成 NFO 改为 toggle。
- 删除：
  - 自动刷新间隔。
  - 追更自动下载到本地。
  - NFO 输出目录。
  - 默认本地媒体根目录。
  - 下载执行方式单选。
  - 唯一匹配自动下载。
  - 按优先级选择。
- 媒体目录规则固定为一个挂载根目录 `media`，自动生成：
  - `media/anime`
  - `media/movies`
  - `media/tv`
- 命名模板分类型：
  - 动画模板。
  - 电影模板。
  - 电视剧模板。
- 自动选择规则分类型：
  - 动画。
  - 电影。
  - 电视剧。
- 下载器设置改为优先级列表：
  - 可添加多个实例。
  - 支持类型：`pikpak rclone`、`pikpak api`、`aria2`、`qb`。
  - 可拖动排序。
  - 单个资源下载失败 3 次后切换下一个下载器。
  - 全部失败后进入需用户处理状态。
- PikPak 远端目录放到对应 PikPak 下载器实例配置里。

## Public Interfaces / Data Contract

- 删除旧导入接口，不保留兼容。
- 新增 RSS 订阅配置接口：
  - `GET /api/rss-subscriptions`
  - `POST /api/rss-subscriptions`
  - `PUT /api/rss-subscriptions/{id}`
  - `DELETE /api/rss-subscriptions/{id}`
- 新增通用媒体页面接口：
  - `GET /api/media/{type}`
  - `GET /api/media/{type}/{entry_id}`
  - `PUT /api/media/{type}/{entry_id}`
- 新增集数资源接口：
  - `GET /api/entries/{entry_id}/episodes`
  - `PUT /api/episodes/{episode_id}/resource`
  - `PUT /api/episodes/{episode_id}/subtitle`
- 设置接口继续使用现有 `/api/settings`，但 payload 改为新字段；旧字段不在 UI 展示。
- `PLAN.md` 全量替换为本计划，不再保留旧“云盘长期媒体库/同步缓存”主线描述。

## Test Plan

- 前端构建：
  - `package-clean.bat` 必须成功。
  - 产物里 `deploy-nas.sh` 保持 LF。
- 导航验证：
  - 左侧没有“导入”。
  - 左侧有“日志”。
  - 新番/番剧/电影/电视剧/设置页面能切换。
- 控制台验证：
  - 未选择队列时显示空状态。
  - 队列详情无旧筛选按钮。
  - 日志和维护不再出现在控制台。
- 新番验证：
  - 卡片按季/篇章显示，不折叠。
  - 卡片显示 `可观看 N 集` 和最近更新状态。
  - 高级筛选默认收起。
  - 添加 RSS 弹窗可保存 Mikan 类型订阅。
- 详情验证：
  - 无旧动作按钮。
  - `信息` 和 `集数资源` Tab 正常显示。
  - 归档后新番页隐藏，番剧页仍可见。
- 设置验证：
  - 旧自动下载/自动刷新/NFO 输出目录/系统 Tab 不显示。
  - 下载器列表可新增、删除、排序、保存。
  - 三类媒体命名模板可保存。
- 后端验证：
  - 旧导入 API 返回 404。
  - RSS 扫描仍能生成条目和集数资源。
  - 下载成功只更新集数资源的已下载状态。
  - 任务失败只显示在控制台任务/日志，不影响媒体详情数据。

## Assumptions

- 本轮不实现真实旧番/电影/电视剧导入，只做页面、向导骨架和数据入口预留。
- 不兼容旧导入 API 和旧导入页面。
- 新番是动画条目的一个状态，不是独立媒体类型。
- 番剧库显示所有动画，包括归档后的新番。
- PikPak 只是下载器之一，不再作为媒体数据核心。
- 媒体根目录固定为容器内 `media`，不存在子目录时自动创建。
- 实现会按阶段提交；涉及测试部署前运行 `package-clean.bat`。
